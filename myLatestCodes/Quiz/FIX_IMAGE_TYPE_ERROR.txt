FIXING THE "Unknown column 'q.image_type'" ERROR
==============================================

This error occurs when the database table is missing the image_type column.

QUICK FIX OPTION 1: Update Existing Database
=============================================
If you already have a database with existing data:

1. Open phpMyAdmin: http://localhost/phpmyadmin
2. Select the quiz_db database
3. Copy and run this SQL command:

   ALTER TABLE questions ADD COLUMN image_type VARCHAR(50) DEFAULT 'image/jpeg' AFTER image_data;

Or use the migrate.sql file:
1. Open phpMyAdmin
2. Select quiz_db database
3. Click "Import" tab
4. Select migrate.sql file from your Quiz folder
5. Click "Go"

QUICK FIX OPTION 2: Fresh Database (Recommended)
==============================================
If you don't have important data yet:

1. Open phpMyAdmin: http://localhost/phpmyadmin
2. Delete the quiz_db database (drop it)
3. Import the updated database.sql file:
   - Click "Import" tab
   - Select database.sql from your Quiz folder
   - Click "Go"
4. Go to http://localhost/myLatestCodes/Quiz/upload_question.php to add questions

The corrected database.sql now includes the image_type column.

VERIFY THE FIX
==============
1. Run the quiz: http://localhost/myLatestCodes/Quiz/
2. If you see "No questions available" - that's correct, just add questions via upload page
3. If you get the image_type error, the column wasn't added properly

If still having issues:
- Check phpMyAdmin > quiz_db > questions table structure
- Verify it shows: id, question_text, image_data, image_type, created_at columns
- All columns should be present for the quiz to work
